## 1. 概要とビジョン

### 1.1 このドキュメントの目的

本ドキュメントは、Tempo AI の UI/UX 設計を記述するものです。デザイナー、開発者、プロダクトマネージャーが共通理解を持ち、一貫性のある体験を作り上げるための指針となります。

**記載内容**:

- 画面のレイアウト構成
- 要素の配置と優先度
- 視覚的な表現方法（文字の大小、余白、色）
- ユーザーの動線と画面遷移
- インタラクション（タップ、スワイプなど）
- トーンと言葉の表現

**記載しない内容**:

- 技術実装の詳細（コンポーネント名、API 仕様など）
- 具体的なピクセル数値
- コードレベルの指定
- データ算出ロジック（プロダクト仕様書を参照）

---

### 1.2 Tempo AI のビジョン

#### コンセプト

**「あなたのテンポで、健やかな毎日を」**

このコンセプトは、以下の価値観を体現しています：

- **個人のリズムを尊重する**: 万人に同じアドバイスをするのではなく、その人固有の生体データ、生活リズム、関心ごとに基づいたパーソナライズ
- **健やかさ**: 医療行為ではなく、日常的なウェルネスのサポート
- **毎日の継続**: 一度きりのイベントではなく、毎日使い続けたくなる体験
- **寄り添い**: 押し付けではなく、優しく見守り、励ます存在
- **環境適応**: 移動や時差があっても、自分のリズムを整えるサポート

#### ユーザー体験の核心

Tempo AI を使うユーザーは、以下のような体験をします：

1. **朝起きたとき**: 「今日の自分」を知り、その日の過ごし方の指針を得る
2. **日中**: ふとアプリを開くと、「様子を見に来てくれた」ような短い声かけを受ける
3. **夜**: 今日を労い、明日への準備を促される
4. **毎日**: 小さな変化を重ねることで、自分の体調の波を理解し、コントロール感を得る
5. **移動中**: 環境が変わっても、リズムを整えるための具体的な指針を得る

#### 「優しいお姉さん」キャラクター

Tempo AI のトーンは「年上の落ち着いた優しいお姉さん」です。このキャラクターは：

- **温かく励ます**: 否定せず、前向きに
- **落ち着いている**: 焦らせない、穏やか
- **知的**: データに基づいた根拠を示す
- **親しみやすい**: 名前で呼びかけ、距離を縮める
- **押し付けない**: 提案はするが、強制しない

### 1.3 適用する UX 心理原則

本アプリは、以下の UX 心理原則を重視します：

#### 認知と心理効果

- **Aesthetic-Usability Effect（美的ユーザビリティ効果）**: 美しいデザインで使いやすさを感じさせる
- **Miller's Law（ミラーの法則）**: 一度に表示する情報は 5-7 項目まで
- **Peak-End Rule（ピーク・エンド則）**: オンボーディング完了時、アドバイス受取時に良い体験を

#### インタラクションとフィードバック

- **Doherty Threshold（ドハティの閾値）**: 0.4 秒以内の反応でストレスを感じさせない
- **Labor Illusion（労力の錯覚）**: AI 分析中を明示し、価値を感じさせる
- **Microinteractions（マイクロインタラクション）**: 細かなアニメーションで体験を豊かに

#### 情報アーキテクチャ

- **Von Restorff Effect（孤立効果）**: 重要な要素を目立たせる
- **Progressive Disclosure（段階的開示）**: 詳細は「もっと見る」で展開

#### ヘルス＆ウェルネスアプリ固有の原則

- **Data Transparency（データの透明性）**: データソースを明示
- **Positive Reinforcement（ポジティブな強化）**: 小さな成功を祝う
- **Contextual Recommendations（文脈に応じた提案）**: 時間帯、天気、状況に応じたアドバイス

## 2. デザイン原則

### 2.1 Tempo AI 固有のデザイン原則

#### 原則 1: 情報の優先順位を明確にする

一度に多くの情報を見せすぎない。最も重要な情報を最初に、詳細は必要に応じて展開。

**実践方法**:

- ホーム画面では「今日のアドバイス」を最優先
- メトリクスは 4 つに絞る
- 詳細画面でのみ深い情報を提供
- コンディション画面で客観的なデータを整理して表示

#### 原則 2: データを「感じさせる」

数値だけでなく、視覚的・言葉的に「今日の自分」を感じられるようにする。

**実践方法**:

- スコアだけでなく「良好な回復」など状態を言葉で表現
- プログレスバーで視覚的に理解しやすく
- AI が生成する文章で、データの意味を解釈して伝える
- コンディション画面でトレンドグラフを表示

#### 原則 3: 押し付けない、寄り添う

アドバイスは提案であり、命令ではない。ユーザーが選択できる余地を残す。

**実践方法**:

- 「〜してみてください」「〜するといいですね」という柔らかい語尾
- 「今日のトライ」は提案であり、強制ではない
- ユーザーが無視しても責めない
- トラベルモード時は「焦らず」「余白を大切に」という安心感を与える

#### 原則 4: 継続を支える

毎日開きたくなる、小さな楽しみや発見がある。

**実践方法**:

- 追加アドバイス（昼・夕）で「見に来てくれた」感
- 今日のトライで新しい発見
- 週のトライで中期的な目標
- コンディション画面で自分の変化を可視化

#### 原則 5: 美しさと機能の両立

見た目の美しさが使いやすさを感じさせる（Aesthetic-Usability Effect）。

**実践方法**:

- 統一されたカラーパレット
- 適度な余白
- 柔らかい角丸、控えめな影
- アニメーションで滑らかな体験

## 3. カラーシステム

### 3.1 カラーの選定コンセプト

Tempo AI のカラーは「自然で穏やかなウェルネス」をコンセプトとしています。

**イメージ**:

- 朝の柔らかい光
- 深呼吸したときの心地よさ
- 自然の中で感じる静けさと活力

---

### 3.2 カラーパレット

#### Primary Color: Soft Sage Green（柔らかいセージグリーン）

**意味**: 成長、調和、健やかさ

**用途**:

- メインの UI 要素（ボタン、アイコン、リンク）
- メトリクスのプログレスバー
- 重要な見出し
- カードの枠線（薄く使用）

#### Secondary Color: Warm Beige（温かみのあるベージュ）

**意味**: 安心感、温もり、落ち着き

**用途**:

- 背景色（全体の背景）
- カードの背景（セクション区切り）
- 今週のトライなど、特定のセクションの背景

#### Accent Color: Soft Coral（柔らかいコーラル）

**意味**: 活力、温かさ、注目

**用途**:

- CTA ボタン
- 重要な通知やバッジ
- 今日のトライなど、注目してほしい要素
- トラベルモードのアイコン

---

### 3.3 テキストカラー

| 種類           | 用途                                   |
| -------------- | -------------------------------------- |
| Primary Text   | 本文、見出し、主要なテキスト           |
| Secondary Text | 補足情報、キャプション、日付など       |
| Tertiary Text  | プレースホルダー、非活性状態のテキスト |

---

### 3.4 背景カラー

| 種類               | 用途                                             |
| ------------------ | ------------------------------------------------ |
| Primary Background | アプリ全体の背景（オフホワイト）                 |
| Card Background    | カード、モーダル、重要なコンテンツエリア（純白） |

---

### 3.5 状態カラー

| 種類                   | 用途                                       |
| ---------------------- | ------------------------------------------ |
| 注意・警告（オレンジ） | トラベルモード時の「調整中」表示           |
| 良好（グリーン）       | 「安定」「良好」などのポジティブな状態表示 |

## 8. トラベルモード

### 8.1 トラベルモードの概要

**コアコンセプト**: 「環境が変わっても、自分のリズム（Tempo）を整える」

**特徴**:

1. リズムのずれに気づかせる
2. リセット方法を教える（光、CT、カフェイン）
3. 焦らせない（適応期間という安心感）

## 14. トーン・表現方法

### 14.1 「優しいお姉さん」の体現

#### 呼びかけ方

- ニックネームを使用: 「マサさん、おはようございます」

#### 語尾

| パターン               | 例                                       |
| ---------------------- | ---------------------------------------- |
| 「〜ですね」           | 「良質な睡眠が取れましたね」             |
| 「〜してみてください」 | 「深呼吸を 3 回してみてください」        |
| 「〜するといいですね」 | 「23 時までに就寝できるといいですね」    |
| 「〜かもしれませんね」 | 「少し疲れを感じやすいかもしれませんね」 |

#### 励まし方

- ポジティブな言葉: 「良好」「順調」「素晴らしい」
- データを根拠に
- 否定しない: 「悪い」ではなく「やや低め」

---

### 14.2 時間帯別の挨拶

| 時間帯          | 挨拶                   |
| --------------- | ---------------------- |
| 朝（6-12 時）   | 「おはようございます」 |
| 昼（13-18 時）  | 「こんにちは」         |
| 夕（18 時以降） | 「お疲れさまです」     |

---

### 14.3 トラベルモード時の特別なトーン

- 「体内時計がまだ調整中ですね」
- 「あと 2-3 日で体が慣れてきます」
- 「焦らず、余白を大切に」
- 「移動による一時的な変化です」
