## 1. 概要とビジョン

### 1.1 このドキュメントの目的

本ドキュメントは、Tempo AI の UI/UX 設計を記述するものです。デザイナー、開発者、プロダクトマネージャーが共通理解を持ち、一貫性のある体験を作り上げるための指針となります。

**記載内容**:

- 画面のレイアウト構成
- 要素の配置と優先度
- 視覚的な表現方法（文字の大小、余白、色）
- ユーザーの動線と画面遷移
- インタラクション（タップ、スワイプなど）
- トーンと言葉の表現

**記載しない内容**:

- 技術実装の詳細（コンポーネント名、API 仕様など）
- 具体的なピクセル数値
- コードレベルの指定
- データ算出ロジック（プロダクト仕様書を参照）

---

### 1.2 Tempo AI のビジョン

#### コンセプト

**「あなたのテンポで、健やかな毎日を」**

このコンセプトは、以下の価値観を体現しています：

- **個人のリズムを尊重する**: 万人に同じアドバイスをするのではなく、その人固有の生体データ、生活リズム、関心ごとに基づいたパーソナライズ
- **健やかさ**: 医療行為ではなく、日常的なウェルネスのサポート
- **毎日の継続**: 一度きりのイベントではなく、毎日使い続けたくなる体験
- **寄り添い**: 押し付けではなく、優しく見守り、励ます存在

#### ユーザー体験の核心

Tempo AI を使うユーザーは、以下のような体験をします：

1. **朝起きたとき**: 「今日の自分」を知り、その日の過ごし方の指針を得る
2. **日中**: コンディション画面で自分の状態を客観的に把握
3. **毎日**: 小さな変化を重ねることで、自分の体調の波を理解し、コントロール感を得る

#### 「優しいお姉さん」キャラクター

Tempo AI のトーンは「年上の落ち着いた優しいお姉さん」です。このキャラクターは：

- **温かく励ます**: 否定せず、前向きに
- **落ち着いている**: 焦らせない、穏やか
- **知的**: データに基づいた根拠を示す
- **親しみやすい**: 名前で呼びかけ、距離を縮める
- **押し付けない**: 提案はするが、強制しない

### 1.3 適用する UX 心理原則

本アプリは、以下の UX 心理原則を重視します：

#### 認知と心理効果

- **Aesthetic-Usability Effect（美的ユーザビリティ効果）**: 美しいデザインで使いやすさを感じさせる
- **Miller's Law（ミラーの法則）**: 一度に表示する情報は 5-7 項目まで
- **Peak-End Rule（ピーク・エンド則）**: オンボーディング完了時、アドバイス受取時に良い体験を

#### インタラクションとフィードバック

- **Doherty Threshold（ドハティの閾値）**: 0.4 秒以内の反応でストレスを感じさせない
- **Labor Illusion（労力の錯覚）**: AI 分析中を明示し、価値を感じさせる
- **Microinteractions（マイクロインタラクション）**: 細かなアニメーションで体験を豊かに

#### 情報アーキテクチャ

- **Von Restorff Effect（孤立効果）**: 重要な要素を目立たせる
- **Progressive Disclosure（段階的開示）**: 詳細は「もっと見る」で展開

#### ヘルス＆ウェルネスアプリ固有の原則

- **Data Transparency（データの透明性）**: データソースを明示
- **Positive Reinforcement（ポジティブな強化）**: 小さな成功を祝う
- **Contextual Recommendations（文脈に応じた提案）**: 時間帯、天気、状況に応じたアドバイス

## 2. デザイン原則

### 2.1 Tempo AI 固有のデザイン原則

#### 原則 1: 情報の優先順位を明確にする

一度に多くの情報を見せすぎない。最も重要な情報を最初に、詳細は必要に応じて展開。

**実践方法**:

- ホーム画面では「今日のアドバイス」を最優先
- コンディション画面では 4 つのメトリクスに絞る
- 詳細画面でのみ深い情報を提供

#### 原則 2: データを「感じさせる」

数値だけでなく、視覚的・言葉的に「今日の自分」を感じられるようにする。

**実践方法**:

- スコアだけでなく「良好な回復」など状態を言葉で表現
- プログレスバーで視覚的に理解しやすく
- AI が生成する文章で、データの意味を解釈して伝える
- コンディション画面でトレンドグラフを表示

#### 原則 3: 押し付けない、寄り添う

アドバイスは提案であり、命令ではない。ユーザーが選択できる余地を残す。

**実践方法**:

- 「〜してみてください」「〜するといいですね」という柔らかい語尾
- 「今日のトライ」は提案であり、強制ではない
- ユーザーが無視しても責めない

#### 原則 4: 継続を支える

毎日開きたくなる、小さな楽しみや発見がある。

**実践方法**:

- 今日のトライで新しい発見
- コンディション画面で自分の変化を可視化
- 朝のアドバイスが楽しみになる質の高さ

#### 原則 5: 美しさと機能の両立

見た目の美しさが使いやすさを感じさせる（Aesthetic-Usability Effect）。

**実践方法**:

- 統一されたカラーパレット
- 適度な余白
- 柔らかい角丸、控えめな影
- アニメーションで滑らかな体験

## 3. カラーシステム

### 3.1 カラーの選定コンセプト

Tempo AI のカラーは「自然で穏やかなウェルネス」をコンセプトとしています。

**イメージ**:

- 朝の柔らかい光
- 深呼吸したときの心地よさ
- 自然の中で感じる静けさと活力

---

### 3.2 カラーパレット

#### Primary Color: Soft Sage Green（柔らかいセージグリーン）

**意味**: 成長、調和、健やかさ

**用途**:

- メインの UI 要素（ボタン、アイコン、リンク）
- メトリクスのプログレスバー
- 重要な見出し
- カードの枠線（薄く使用）

#### Secondary Color: Warm Beige（温かみのあるベージュ）

**意味**: 安心感、温もり、落ち着き

**用途**:

- 背景色（全体の背景）
- カードの背景（セクション区切り）

#### Accent Color: Soft Coral（柔らかいコーラル）

**意味**: 活力、温かさ、注目

**用途**:

- CTA ボタン
- 重要な通知やバッジ
- 今日のトライなど、注目してほしい要素

---

### 3.3 テキストカラー

| 種類           | 用途                                   |
| -------------- | -------------------------------------- |
| Primary Text   | 本文、見出し、主要なテキスト           |
| Secondary Text | 補足情報、キャプション、日付など       |
| Tertiary Text  | プレースホルダー、非活性状態のテキスト |

---

### 3.4 背景カラー

| 種類               | 用途                                             |
| ------------------ | ------------------------------------------------ |
| Primary Background | アプリ全体の背景（オフホワイト）                 |
| Card Background    | カード、モーダル、重要なコンテンツエリア（純白） |

---

### 3.5 状態カラー

| 種類             | 用途                                       |
| ---------------- | ------------------------------------------ |
| 良好（グリーン） | 「安定」「良好」などのポジティブな状態表示 |
| 普通（イエロー） | 「普通」の状態表示                         |
| 注意（オレンジ） | 「やや低下」の状態表示                     |

### 6.2 構成要素の説明

#### ヘッダー

- 時間帯別の挨拶 + ニックネーム
- 日付 + 天気アイコン

#### 今日のアドバイスカード（ホーム画面表示）

- AI 生成のパーソナライズされた文章

- **condition.detail**: 8-12 文の詳細説明

- アイコン + タイトル（**daily_try.title**）
- 1 文の概要（**daily_try.summary**）
- 「詳しく見る」リンク

#### 今日のトライ詳細画面

- **daily_try.detail**: 5-7 文の詳細説明
- 「なぜ今日これなのか」の理由を含む
- ユーザー固有の相関データがあれば活用

#### お見送りメッセージ

- 画面下部に表示
- 温かい締めの一言

### 7.2 構成要素

#### 24時間サークル図 + HRV統合表示

- **外周**: 24時間の時刻を表示（時計型）
- **睡眠帯**: 昨夜の睡眠時間を弧で塗りつぶし
- **過去7日のプロット**: 薄い点で就寝・起床時刻を表示
- **中央**: HRV値（大きく）+ 7日平均との差分
- **下部テキスト**: 昨夜の就寝時刻・今朝の起床時刻
- **タップ**: 詳細画面へ遷移

#### リズム安定度

- **ステータス**: 良好 / やや不安定 / 不安定
- **インジケーター**: ●●●○○（5段階）
- **説明文**: リズムの状態と回復への影響を1行で表示
- **背景色**: ステータスに応じて変化（良好=薄緑、やや不安定=薄黄、不安定=薄オレンジ）

#### 要因マップ

各要因の表示構成:
- **ラベル**: 睡眠 / 環境 / 活動
- **プログレスバー**: 10段階（貢献度を視覚化）
- **貢献度テキスト**: 「回復に大きく貢献」など
- **詳細テキスト**: 具体的な数値（睡眠時間、気圧、歩数など）

#### AIの見立て

- **内容**: 3-5文の統合解釈
- **トーン**: 優しいお姉さん（他のAI生成テキストと統一）
- **ボタン**: 「詳しく見る」で詳細画面へ遷移

## 13. トーン・表現方法

### 13.1 「優しいお姉さん」の体現

#### 呼びかけ方

- ニックネームを使用: 「マサさん、おはようございます」

| パターン               | 例                                       |
| ---------------------- | ---------------------------------------- |

### C. v2.x からの主な変更点

**削除された要素:**

1. **ホーム画面のメトリクスカード**: コンディション画面との重複解消
2. **追加アドバイス（昼・夕）**: シンプル化
3. **今週のトライ**: 今日のトライに集中
4. **ストレスグラフ**: HRV に統合
5. **環境詳細画面**: 簡易表示のみ

**変更された点:**

1. **ホーム画面**: アドバイス + トライのみ
2. **コンディション画面**: 4 メトリクス + 環境情報（1 行）
3. **詳細画面**: 7 画面 → 4 画面

**理由:**

- 情報過多の回避
- 役割分担の明確化（ホーム=アドバイス、コンディション=データ）
- 開発効率の向上

---

### D. v3.0 からの変更点（v3.1）

**変更された要素:**

1. **action_suggestions 削除**: 「具体的な行動提案」セクションを削除
2. **condition.detail 拡張**: 行動提案を自然な文章に織り込む形式に変更
3. **ホーム画面のレイアウト調整**: サマリー + 詳細画面の 2 段階表示に

**理由:**

- action_suggestions と Daily Try の重複解消
- より自然な文章でアドバイスを提供
- トークン効率の向上

---

### E. v3.1 からの変更点（v3.2）

**削除された要素:**

1. **4つの小カード（睡眠、HRV、活動量、ストレス）**: 要因マップに統合
2. **ストレスメトリクス**: HRVに統合
3. **5つの詳細画面**: 1画面に統合
4. **環境情報の1行表示**: 要因マップの「環境」に統合

**追加された要素:**

1. **24時間サークル図 + HRV統合表示**: リズムと自律神経の関係を視覚化
2. **リズム安定度表示**: リズムが「土台」であることを強調
3. **要因マップ**: コンディションの因果関係を可視化
4. **AIの見立て**: 因果関係を言語化
5. **あなたのパターン（散布図）**: ユーザー固有の相関を可視化

**理由:**

- 情報量の削減とシンプル化
- 因果関係の可視化（HRVを結果、その他を要因として構造化）
- サーカディアンリズムの重要性を強調
- 開発コストの削減（5画面→1画面）
